# Paw Studio Growth Agents

Updated: 2026-05-08

## Purpose

Create a local, token-efficient growth workflow for the Paw Studio Apps in Toss
launch. The goal is short-form ads and SNS operations, not autonomous public
posting.

## Control Flow

1. Hermes owns durable learning, Honcho recall, Paperclip checklists, and
   Ouroboros self-check habits.
2. Luna/OpenClaw receives Telegram and local commands, keeps routine work on
   Ollama Cloud, and delegates only summarized briefs.
3. ShortsMaker drafts scripts, shot lists, edit checklists, and media prompts.
4. SNSManager builds posting calendars, captions, hashtags, upload checklists,
   and analytics summaries.
5. Star/Codex is called for automation code, repo work, upload tooling, or
   integration fixes.
6. Solar/Claude is called for launch strategy, policy risk, reputation risk, and
   careful review.
7. Gemini/image-video tools are called only when actual image/video generation
   or multimodal inspection is needed.

## Approval Gates

Human approval is required before:

- public posting or deleting
- customer DMs or comments
- ad spend
- changing production app settings
- uploading private user images to external services
- connecting new SNS credentials

## Paperclip Checklists

Paperclip should hold the recovery state so a compressed or restarted session
does not lose direction.

- Campaign objective
- Target persona
- Offer or hook
- Creative variants
- Required assets
- Platform destinations
- Approval state
- Publish status
- Results and lessons

## Ouroboros Self-Check

Use Ouroboros explicitly for recurring campaign templates and automation rules:

- clarify acceptance criteria
- check whether the plan can be tested locally
- list failure modes before publishing
- record the prevention rule after a mistake

Keep Ouroboros single-worker and low-turn by default.

## Default Task Split

- Luna: coordinator, Telegram, local execution, cheap routing.
- Hermes: memory and learning loop.
- ShortsMaker: creative production brief.
- SNSManager: calendar, captions, upload checklist, analytics.
- Star: Codex implementation and automation.
- Solar: Claude review and risk analysis.
